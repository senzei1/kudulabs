End-to-End CDP Pipelines with Apache Kudu

Hands-on labs that wire together NiFi, Kafka, Kudu, Impala, Spark, Ranger, and Iceberg into production-grade architectures. Built for engineers who learn by building.

Each lab walks through a real-world scenario — from raw data ingestion to analytical queries — using multiple CDP components. Every step includes the actual commands, SQL, NiFi processor configurations, and Spark code you need to build it yourself.

Lab 01

Real-Time Telco CDR Pipeline

Ingest millions of Call Detail Records via NiFi, stream through Kafka, land in Kudu with hash+range partitioning, enrich with fraud scoring, and query interactively through Impala. Includes regulatory retention via partition lifecycle management.

NiFi Kafka Kudu Impala Hue
Lab 02

Continuous Churn Prediction

Build a real-time feature store in Kudu that aggregates behavioral signals from NiFi streams. Score subscribers continuously with Spark ML. Serve churn risk tiers to retention dashboards via Impala. Closed-loop: campaign responses feed back into the feature store.

NiFi Kudu Spark ML Impala Hue
Lab 03

CDC Inventory & Warehouse Tracking

Capture changes from a PostgreSQL inventory database via NiFi CDC, replicate to Kudu for real-time stock visibility, run replenishment queries in Impala, and age historical data to Iceberg via Spark for long-term analytics. Full tiered architecture.

NiFi CDC Kudu Impala Spark Iceberg
Lab 04

Compliance Audit Platform

Ingest regulatory filings via NiFi into Kudu tables with immutable audit columns. Apply column-level security with Ranger. Track review workflows with mutable status fields. Demonstrate tamper-evident design patterns for LGPD, CRC, and IFT requirements.

NiFi Kudu Ranger Impala

Environment Requirements

  • CDP Private Cloud Base 7.1.9+ or CDP Public Cloud Data Hub
  • Services: HDFS, Kudu, Impala, NiFi (or CDF), Kafka, Spark, Ranger, Hue
  • Kudu 1.17.0+ (required for immutable columns, auto-incrementing, per-range hash schemas)
  • Impala with Kudu integration enabled (default in CDP)
  • Admin access to NiFi canvas for processor deployment
  • Ranger admin access for Lab 04 (policy configuration)
  • A PostgreSQL instance for Lab 03 (CDC source — can be a Docker container)

These labs are designed to be completed in order but each is self-contained. Lab 01 establishes the foundational NiFi → Kafka → Kudu → Impala pattern that the other labs build on.
